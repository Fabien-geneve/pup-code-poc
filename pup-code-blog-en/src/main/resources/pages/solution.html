Intentions

This page give the vision i have for a peer-to-peer exchange system. I came from few month ago, while i turned in my mind, the privacies problem that's Social Network have. It don't come in a one day, it is a long reflection on the why, and on the how achieve a solution.

A Peer Definition                                                                                       Summary


  • A Peer is a People, who have an identity, that the system must preserve.
  • A Peer can have multiple devices, or just one, who can be: Smartphone, Tablet, home Computer or Laptop.
  • Each device own an Operating System. For each Operating System, un "PUP Code Agent" need to be installed.
  • Each device have one or more Internet Interface: phone service data like 3/4 G, wifi, Bluetooth.
  • Each device of each peer, contains a PUP Code Agent to dialog with other devices, the other of the peer, on each peer containing in its network.
With these characteristics, each Peer of the System, can communicate directly with the other Peers of the "Friends Network", on making TCP/IP connection on the other devices of them friends. 

But for having this feature, each active device, need to have the configuration IP of the other devices to make connections. Even some devices have multiple Internet Interface. 

So the agent installed on each devices should be aware of the interface change event. 

A new IT Model, finding its inspiration in the P2P model                   Summary



The coordination server store only the configuration of each "Network of friends", anonymously, listen of each networks devices configuration change event, and push the configuration to each device of the Network.

It is the same mechanism than torrent file with Peer exchange Server. It give the TCP/IP configuration of the peer of the network having the desired resource. 

In our case, the Coordination Server will listen the change IP event, relayed by the agents installed on the devices. And will push the configuration of the updated Network configuration to the other devices of the network.

About the security                                                                                     Summary


Some ideas to ensure the security of the system : 
  • Communication of devices with the coordination server could be certificate based.
  • Set up filters for DDoS attack and ensure redundancy.
  • Encryption of data, with a specific key pair certificate by "Friends Network". We can imagine a certification authority, with a root trust certificate, and the system generate certificates and serve these certificate to each peer, to have a based certificate dialog by Network.
